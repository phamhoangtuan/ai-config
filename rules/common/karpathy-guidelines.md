# Karpathy Guidelines

Use the `karpathy-guidelines` skill for coding work that writes, edits, reviews, debugs, or refactors code.

When the task involves implementation or code review:
- Read `skills/karpathy-guidelines/SKILL.md` before making code changes or giving a code-review judgment.
- Apply the four principles from that skill while working.

Scope the skill narrowly:
- Apply it for code-writing, bug-fixing, refactoring, and code-review tasks.
- Do not apply it for planning-only, brainstorming-only, documentation-only, or repo-administration tasks unless they include substantive code changes or code review.

This rule is always-on for Claude Code because files in `rules/common/` are loaded automatically.
OpenCode receives the skill through `skills/`, but does not consume this rule.
